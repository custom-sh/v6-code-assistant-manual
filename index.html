可视化配置

  1. 1

    打开配置面板

    右键项目 → 打开插件配置；或通过 File → Settings → IMS V6 Code Assistant 进入。面板分为「常规」和「数据库配置」两个页签

  2. 2

    常规设置

    在「常规」页签中配置 IDE 运行环境

    语言：切换插件显示语言（中文 / English / 跟随IDE）

    编码：设置项目文件编码

    项目 SDK：选择项目使用的 JDK 版本

    Maven：配置 Maven Home、User Settings、Local Repository 路径

    编辑器：勾选 Java 软换行、代码格式化开关、自动优化导包

  3. 3

    填写项目信息

    在「项目编码」和「项目名称」输入框中填写实际值。项目编码只能包含字母和数字，用于替换模板中的 demo 占位符

  4. 4

    配置多端路径与端口

    分别设置 Server、Web、PDA 三端的项目路径和端口号。路径框右侧实时显示验证状态（有效/不存在/不是目录），端口号可自定义

  5. 5

    查看版本信息

    每端路径右侧显示平台版本和项目版本。多端版本不一致时弹出警告通知

  6. 6

    配置数据库连接（可选）

    切换到「数据库配置」页签，添加 PostgreSQL / Oracle / DM 等数据库连接，填写主机、端口、库名、用户名、密码

  7. 7

    点击「一键初始化模板」

    填写完项目信息和三端路径后按钮自动启用。点击弹出确认对话框，展示详细步骤说明

  8. 8

    确认初始化操作

    仔细阅读确认对话框中的 10 步操作说明，点击「确定」开始。包括：修改 pom.xml、重命名包和类、创建 Run Configuration、npm install 等

  9. 9

    等待完成

    初始化完成后弹出通知，显示处理文件数量

平台版本升级

  1. 1

    打开升级对话框

    在配置面板中点击「升级版本」按钮

  2. 2

    选择新版本模板目录

    点击路径输入框右侧的浏览按钮，选择新版本的 template 目录。参考路径格式：imsv6/release/template/v6.x.x/

  3. 3

    查看升级类型

    系统会自动检测升级类型

    大版本升级：跨版本升级（如 V6.1 → V6.2），将替换项目文件

    修订版本升级：同版本内升级（如 V6.2.1 → V6.2.4），仅替换 lib 和更新 package.json

  4. 4

    选择升级模块

    勾选需要升级的模块：Server、Web、PDA。每个模块会显示当前版本和目标版本

  5. 5

    预览升级步骤

    在「升级步骤预览」区域查看每个模块将执行的具体操作

  6. 6

    确认执行升级

    点击「确定」按钮，系统会自动备份并执行升级。大版本升级会自动备份，升级失败时自动回滚

  7. 7

    查看升级进度

    升级执行过程中，IDEA 底部进度条会实时显示当前操作（备份、复制模板文件、替换等）。升级完成后弹出通知

项目构建打包

  1. 1

    打开构建对话框

    在配置面板中点击「构建项目」按钮

  2. 2

    选择构建模块

    勾选需要构建的模块：Server、Web、PDA。路径无效的模块自动禁用

  3. 3

    设置项目版本号

    版本号自动读取 Server 端当前版本并 +1（如 1.0.0 → 1.0.1）。鼠标悬浮可查看多端各自当前版本。手动修改版本号后，打包时会强制统一写入多端（含未勾选的模块），确保版本一致

  4. 4

    设置输出目录（可选）

    如需打包产物，点击浏览按钮选择输出位置。留空则只执行 make 构建，不打包

  5. 5

    点击「打包」按钮

    系统将并行弹出独立 cmd 窗口执行 make（不占 IDEA 内存，避免卡死闪退）。进度条实时显示各模块构建状态

    进度提示：实时显示 ✓/✗ 成败标记和进度百分比

    成功判定：通过复制 make.bat 并注入结果标记，在 pause 之前即写入 SUCCESS/FAILURE

    窗口行为：构建成功时 cmd 窗口自动关闭；失败/异常时窗口停留保留 pause 供查看错误

    打包输出：仅 make 成功的模块才打包。Server → lib.zip，Web → web.zip，PDA → pda.zip

    通知链接：通知中提供「打开输出目录」可点击链接；全部成功时自动用资源管理器打开

  6. 6

    查看构建进度

    构建过程中，IDEA 底部进度条实时显示各模块构建状态和进度百分比。独立 cmd 窗口并行执行 make，构建完成后弹出通知

代码扫描

  1. 1

    配置扫描规则

    打开插件配置面板并切换到「代码扫描」页签，按需启用规则、设置问题级别和编辑器高亮方式

  2. 2

    选择扫描范围

    在项目视图中选中文件或目录，也可以在编辑器中打开目标文件；未指定目标时将扫描整个项目。支持多选文件，测试源码默认不参与扫描

  3. 3

    执行代码扫描

    右键 → IMS V6 Code Assistant → 代码扫描，系统仅执行当前已启用的 IMS V6 规则

  4. 4

    查看扫描结果

    扫描完成后在 IDEA 原生 Code Inspection 结果窗口中按严重级别查看问题，点击结果可直接定位到对应代码

  5. 5

    修复发现的问题

    根据问题说明修改代码；支持快捷修复的规则可在问题位置使用 IDEA Quick Fix 自动处理

  6. 6

    切换实时扫描

    通过右键菜单中的「启用实时扫描 / 关闭实时扫描」控制编辑器即时检查。关闭实时扫描不会影响手动执行代码扫描

代码生成器

  1. 1

    打开代码生成器

    右键 → IMS V6 Code Assistant → 代码生成器

  2. 2

    粘贴表结构 TSV

    填写「系统编码」「创建人」「版本编码」，粘贴设计文档前 4 列。插件自动识别表名（括号内英文）和表描述（括号前中文），从 TSV 文末 pk_/uk_/ix_ 声明或第 5 列 isKey 标记回填主键与索引栏（仅填空栏，不覆盖已填内容）

  3. 3

    核对索引与操作

    索引栏自动回填后检查主键、唯一索引、普通索引、常用查询条件是否正确。按需勾选附件 / 提交 / 确认等操作

  4. 4

    点「确定」触发后台生成

    对话框不关闭，后台 Task 启动；表单保留，预览取消后可继续微调

    字段去重：自动识别已激活 DB 连接读取库表字段，同时扫描既有 *ColumnDescDef.java，双重跳过

    STAT 翻译：先查项目内 *ColumnType.java + 平台内置字典；未命中走 AI 批量翻译并缓存预翻译表

    占位常量名：AI 仍未命中则回填「字段编码_STAT_序号」，预览窗黄色高亮提示

  5. 5

    预览窗微调

    原生 IDEA Editor 展示 6 个产物（ColumnDescDef/ColumnType/Selector/Table/Model/Vue）。填表窗自动隐藏，可在预览窗直接修改后确定

  6. 6

    确认写盘

    Server 端文件按位置插入到既有类体内（不覆盖整文件），Vue 写入 web 模块。通知显示新增/跳过字段数与命中来源（DB/本地/AI预翻译）

常量同步 & 图标库

  1. 1

    打开 Java 常量文件

    在编辑器中打开 ColumnType.java 或 CheckType.java

  2. 2

    选择同步方式

    ColumnType 同步：右键 → Sync ColumnType to Frontend，将常量同步至前端 columnType.js

    CheckType 同步：右键 → Sync CheckType to Frontend，将检查标识同步至前端 checkType.js

    增量同步：选中部分常量文本后右键执行，仅同步选中的内容

    自定义同步：在弹窗中手动输入需要同步的字段名

  3. 3

    确认同步

    系统显示将新增的常量数量，确认后增量写入。不会覆盖前端已有定义，仅新增缺失常量

  4. 4

    打开图标库

    右键 → IMS V6 Code Assistant → 系统内置图标库

  5. 5

    搜索图标

    在搜索框中输入关键词进行搜索。支持中文关键词搜索，多个关键词用空格分隔（AND 匹配）

  6. 6

    一键复制代码

    点击任意图标，其 pg-icon-xxx 代码会自动复制到剪贴板。复制成功后会显示提示气泡

启动前端 & 切换数据库

  1. 1

    选择前端模块

    在项目视图中右键 → IMS V6 Code Assistant，选择需要启动的服务

    Start Web Server：使用 VS Code 打开 Web 模块并启动开发服务

    Start PDA Server：使用 VS Code 打开 PDA 模块并启动开发服务

    VS Code 不可用：自动回退到独立 cmd 窗口执行启动命令

  2. 2

    信任 VS Code 工作区

    首次打开模块目录时，VS Code 会显示工作区信任提示。确认目录来源可信后，点击「是，我信任作者」；如果继续询问是否允许自动任务，请选择「允许」

  3. 3

    等待自动任务启动

    授权后 VS Code 会读取插件生成的 .vscode/tasks.json，并在终端自动执行任务。没有 node_modules 时先运行 npm install，再执行 npm run serve；已有依赖时直接启动服务

  4. 4

    未自动启动时手动运行

    如果该目录已在 VS Code 中打开，自动任务可能不会再次触发。按 Ctrl+Shift+P，搜索并选择「任务: 运行任务」（Tasks: Run Task），再选择「npm: serve」或「npm install & serve」

  5. 5

    查看启动结果

    终端显示编译成功后，在浏览器中打开对应页面。Web/PDA 端口可在插件配置面板中修改

  6. 6

    配置数据库连接

    在配置面板「数据库配置」页签中添加连接（PostgreSQL / Oracle / DM 等），填写主机、端口、库名、用户名、密码

  7. 7

    切换数据库

    右键 → IMS V6 Code Assistant → Switch Database ▶，展开子菜单查看所有已配置连接。当前激活连接以 🟢 绿色圆点标识，选中即切换并写入 server/application.properties

开发手册问答

  1. 1

    配置 AI 模型

    在配置面板的「AI 模型配置」区域设置 Provider、Model、API Key 和 Endpoint。支持 OpenAI、Claude、DeepSeek 和自定义兼容 API

  2. 2

    配置知识库目录

    在「知识库文档目录」中设置文档路径。支持 Markdown、Java、JS、SQL、XML 等多种格式

  3. 3

    打开问答窗口

    右键 → IMS V6 Code Assistant → AI知识库(beta)，或点击右侧边栏的 V6 Knowledge QA

  4. 4

    开始提问

    在输入框中输入问题，按 Enter 发送。Enter 发送，Shift+Enter 换行。AI 会基于知识库文档进行回答

  5. 5

    管理对话

    使用工具栏按钮清空对话历史或停止生成。AI 回复语言会跟随插件的语言设置自动切换

常见问题

是否支持在同一 IDEA 窗口中打开多个项目？

暂不支持在同一 IDEA 窗口中同时打开或附加多个项目，否则不同项目的插件配置可能互相覆盖。建议每个项目使用独立的 IDEA 窗口打开，确保配置相互隔离。

插件支持哪些 IntelliJ IDEA 版本？

支持 IntelliJ IDEA 2023.1 及以上版本，包括 Ultimate 和 Community 版本。

如何切换插件语言？

在配置面板中可以切换中文/英文，切换后整个插件界面（包括菜单、对话框、通知、手册）都会实时更新。

多端版本不一致怎么办？

配置面板会自动检测多端版本，不一致时弹出警告通知。你可以使用「升级版本」功能统一升级，或在「构建项目」时强制统一版本号。

构建时 IDEA 卡死怎么办？

IMS V6 Code Assistant 使用独立 cmd 窗口执行 make，不占用 IDEA 内存。如果仍然卡顿，请检查项目路径配置是否正确。
