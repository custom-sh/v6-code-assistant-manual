当前插件版本 v1.1.7

更新日志

版本记录

Version 1.1.7

Added

  • Table 类名规范新增 Quick Fix，通过 IDEA Rename 重构同步更新类名、构造方法、文件名和代码引用。
  • Model 类名规范新增 Quick Fix，通过 IDEA Rename 重构同步更新类名、构造方法、文件名和代码引用。
  • CheckDataIntercept、WebDataModelIntercept、WebMasterDaoIntercept 类名规范新增 IDEA Rename Quick Fix。
  • 新增 ColumnType 常量注释规则，检查 util/*ColumnType.java 静态内部类中的常量字段是否添加 Javadoc。
  • 新增多行 SQL formatter 标记保护检查：SQL 前后 10 行内检查 @formatter:off/on 标识。
  • 新增超长单行 SQL 源码行检查：提示按结构拆分。

Changed

  • UK 命名规则支持单个唯一约束使用 uk_表名；同一 Table 存在多个唯一约束时继续使用带后缀的命名。
  • 抽象类命名和二开创建者规则的严重级别迁移，已保存配置自动升级为 TYPO。

Version 1.1.6

Added

  • DataRow/Params 字段键小写规范新增 Quick Fix，可一键将字符串字面量中的字母转换为小写，保留原有字符和分隔符。

Changed

  • Model 类命名规则根据 setModelCode(...) 推导类名；ModelCode 缺少项目编码前缀时按插件项目配置自动补齐，已有前缀时不重复添加。
  • CheckDataIntercept 类命名规则与 Model 类保持一致，ModelCode 缺少项目编码前缀时自动补齐。
  • 代码扫描规则列表新增默认开启的"项目模式"配置；开启时 Model 与 CheckDataIntercept 类名自动包含项目编码，关闭时仅跟随 ModelCode。
  • 右键"编码规约扫描"不再要求预先保存项目配置，可直接执行已启用的扫描规则。
  • 变量和方法 lowerCamelCase 规则允许在小驼峰名称内部使用任意全大写缩写词（如 SL、SMT、URL、HTTP）。

Fixed

  • 修复 Oracle 子查询别名规则误报正常列别名的问题；仅检查 FROM/JOIN (SELECT ...) AS alias 场景。
  • 修复数组中括号位置规则误报 Java 可变参数 Object... args 的问题。
  • 完善 DataRow 字段键规则对 CaseSensitiveDataRowImpl 的排除，支持识别父类型变量通过 create() 或构造方法初始化的场景。

Version 1.1.5

Added

  • 新增基于 IDEA PSI 的代码扫描：支持当前文件、目录、多选文件和整个项目手动扫描，并可按项目开启或关闭编辑器实时检测
  • 内置 59 条通用与IMS V6 开发规范，规则首次安装默认不启用，可按需配置启用状态、严重级别和高亮方式
  • 新增多项快捷修复（Quick Fix）：支持键处理
  • 代码生成器支持在预览中选择需要写入的代码文件，未勾选内容不会写入项目
  • 新增「代码注释模板」设置：创建类时自动生成 Pangus IMS 版权与类注释，一键初始化时可同步更新已有 Java 文件注释

Changed

  • 重构配置面板布局
  • 一键初始化增加注释内容的替换
  • 平台升级前自动检测并提示停止运行中的服务，减少文件占用导致的升级失败
  • 更新内置使用手册，补充代码扫描说明、项目多窗口使用建议及最新功能介绍

Fixed

  • 修复系统图标库连续复制时界面卡顿的问题
  • 修复数据库驱动按需加载、部分国际化显示问题

Version 1.1.0

Added

  • 配置面板新增端口设置、支持一键修改端口

Changed

  • 升级弹窗布局优化：新版本路径标签+输入框+升级类型同行，去掉 TitledBorder 和分割线，紧凑无空白
  • 降级拦截：目标版本 ≤ 当前版本时禁用确定按钮，分情况提示（相同无须升级 / 低于暂不支持）

  • npm 启动服务重构：通用 runNpmCommand 方法，npm 可用性探测
